# Risk Analysis

Use this skill to systematically surface and assess risks so a manager can make informed decisions with eyes open.

## When to use

- A new project, feature, or strategy is being proposed.
- A manager asks "what could go wrong?" about a plan.
- You need a structured assessment before approving a major commitment.

## Input

Provide a description of the initiative, timeline, budget, key stakeholders, and any known constraints.

## Output format

1. **Risk register** - Table with columns: risk, category, likelihood (low/medium/high), impact (low/medium/high), and risk score.
2. **Top risks** - The three risks with the highest combined score, each with a brief mitigation strategy.
3. **Assumptions** - List of assumptions that, if wrong, would change the risk profile.
4. **Recommendation** - One-sentence go/no-go suggestion with the primary reason.

## Guidelines

- Consider financial, operational, reputational, and technical risk categories.
- Be honest about likelihood; do not downplay risks to make the plan look better.
- Suggest concrete mitigations, not just "monitor the situation."
- If insufficient information is provided, list the risks you can assess and flag gaps.
