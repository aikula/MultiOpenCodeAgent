# Task Decomposition

Use this skill when the user asks to break down, decompose or plan tasks.

Return:

1. Main goal statement
2. Subtasks (ordered by dependency)
3. Estimated complexity for each (S/M/L)
4. Dependencies between subtasks
5. Critical path identification
6. Quick wins that can be done immediately
7. Blockers to resolve first

Be practical. Suggest concrete first steps, not abstract planning.
